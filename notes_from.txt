For example, suppose your pipeline works as follows:
You use Gemini Flash to generate alternate phrasings of the user’s question.
You then search for relevant sources based on these alternate phrasings.
Finally, you use GPT to generate the final answer based on the retrieved sources.
In this case, TTFT should be measured up to the first token produced by GPT in the final answer, not the first token produced by Gemini Flash.

However, if instead of using GPT at the end you send parallel requests to multiple models and return the response from whichever model generates the first token first, that is acceptable. Note that the time spent generating alternate phrasings and performing the search must still be included in the TTFT measurement.

Sergey Verentsov — 03/03/2026 19:15
Non-unique files is an error, that will be fixed prior to the release of the public dataset on March 11.

About grounding procedure.
For each question, there is a predefined golden set of sources. This set is verified to be sufficient for generating the correct answer. In this context, a "source" refers to a specific page within a document.
Your predicted set of sources will be compared against the golden source set. The evaluation metric is a weighted F-score with β = 2.5 (see the formula in the attachment). This weighting makes recall ~6 times more important than precision, meaning that missing required evidence is penalized significantly more heavily than including unnecessary sources.

About your other questions: 
If the same info is in 10 different sources, do we need to provide all 10?
 No. You are expected to identify the exact source specified in the golden set. For example, if a question relates to a specific section of a law and the dataset contains multiple editions of that law, the golden set will include only the latest edition - even if the same section appears in earlier versions.
What if there is a page that does not provide a direct answer, but contains secondary, yet still relevant information?
  If the information on a page is not required to generate the reference answer, that page will not be included in the golden set, even if it contains related information.

Sergey Verentsov — 03/03/2026 19:30
An LLM-based judge compares free-text answers to the reference answers.
A penalty is applied, see the formula in my previous message.
The dataset intentionally includes questions that do not have an answer in the corpus.
For deterministic questions (those not of type free_text), the expected answer is null, with an empty reference list.
For free_text questions, the expected answer should in free form  state that the information is not available (e.g., "There is no information on this question") with an empty reference list as well.

anatoly_bobrovsky — 05/03/2026 14:31
Great question — and a fair one! Here's how we approach it:

TTFT is self-reported in telemetry, but we have several layers of protection against falsification:

Code audit — all prize contenders undergo a mandatory code review, where reviewers can reproduce the pipeline and cross-check reported timings against actual execution.
Statistical analysis — we run pattern analysis across all telemetry submissions to flag anomalies that are statistically inconsistent.
Telemetry isn't the only signal — it's a relatively small bonus multiplier on top of grounding and answer quality, which carry far more weight. So gaming timestamps would be a lot of effort for minimal gain, with a real risk of disqualification.

Sergey Verentsov — Yesterday at 12:32
@Igor Tomashevskiy @Ilya Rice 
there is special handling for negative actual and actual = 0, here is the exact code of the evaluator: 
 def _calculate_score(self, prediction: Any, reference: Any) -> float:
        try:
            # Parse both as numbers
            pred_val = float(prediction)
            ref_val = float(reference)

            # Special case for zero to avoid division by zero logic or incorrect tolerance
            if ref_val == 0:
                return 1.0 if pred_val == 0 else 0.0

            # Tolerance of up to 1% of reference value
            # Note: Using abs(ref_val) for tolerance calculation to handle negative numbers
            if abs(pred_val - ref_val) < 0.01 * abs(ref_val):
                return 1.0
            return 0.0
        except (ValueError, TypeError):
            # If parsing failed → 0 points
            return 0.0

Sergey Verentsov — 10/03/2026 16:25
Yesterday, there were several questions that were promised to double-check after the webinar. These are confirmed answers to them:

Q: Do names have aliases, for example “UK” and “United Kingdom”?
A: No. Names do not have aliases. When building the dataset, we aim to avoid ambiguities like this, so questions involving names are designed to have a single, unambiguous expected answer.

Q: Can a numeric answer be a float?
A: Yes. Float values are allowed and fall under the same number type as integers. In JSON there is no practical difference between writing 405351504 and 405351504.0 — both are valid. During evaluation, numeric answers are compared with a ±1% tolerance.

Q: What if an answer begins on one page and continues onto the next — should we provide both pages?
A: Yes, please include both pages. If an answer spans multiple pages, list all of them in retrieved_chunk_pages. In most cases, however, answers are contained within a single page and do not cross page boundaries.

Sergey Verentsov — 14:43
We do have some safeguards in place to limit multi-accounting.

When registering, participants are required to provide a link to their LinkedIn or GitHub profile. We verify that these links correspond to real profiles and the registration is tied to a real individual. This serves as our first layer of protection.

At the same time, we intentionally keep the verification process relatively lightweight. Even with much stricter verification (for example, requiring identity documents), it would still be impossible to fully prevent someone from registering through a friend or another person.

Regarding similar team names: we have seen two cases so far and checked both. One was a duplicate and another case: there are actually two teams. Even if they work at the same company and exchange ideas, that is not considered a problem and is generally a healthy part of research competitions.

More importantly, we do not believe that multi-accounting provides a meaningful competitive advantage in this challenge. You can already see this on the current leaderboard: the participant in first place has made only one submission, while many teams have already made multiple submissions. In practice, the final score is much more dependent on the quality of the system than on the number of attempts
